# 🚀 GUIDE: HOW TO MERGE YOUR GENERATED FRONTEND WITH THIS PROJECT

This guide explains how to integrate the new beautiful frontend pages and layouts generated by your external AI (v0, Bolt.new, Lovable, Claude) into this existing, production-ready codebase.

By following this workflow, you will replace the old interface with the new design while preserving 100% of the working database schemas, Google Gemini integrations, and official KEA rank composite calculations!

---

## 🛠️ Step 1: Back Up Your Current Work
Before copying any files, create a backup of your current working directory, or commit your active changes to Git:
```bash
git add .
git commit -m "Backup before merging new frontend design"
```

---

## 🧩 Step 2: Keep the Core "Brain" Files (Do NOT Overwrite!)
Your new frontend needs to speak to the existing calculation engines, serverless functions, and database links. Ensure you **DO NOT** delete or overwrite these critical files:

1.  **Rank Prediction Engine**: [src/lib/rank-predictor.ts](file:///c:/Users/asus/Downloads/coded/src/lib/rank-predictor.ts)
    *   *Why*: Contains the official KEA formula mapping exam scores and PUC percentages to expected ranks.
2.  **AI Counselor Client**: [src/lib/gemini.ts](file:///c:/Users/asus/Downloads/coded/src/lib/gemini.ts)
    *   *Why*: Handles conversation history, message streams, status indicators, and handles fallback logic between Gemini and OpenRouter.
3.  **Google Gemini Backend**: [api/gemini-chat.ts](file:///c:/Users/asus/Downloads/coded/api/gemini-chat.ts)
    *   *Why*: The Vercel serverless function that securely relays messages to Google's official Gemini endpoint.
4.  **Supabase Client Bindings**: [src/integrations/supabase/client.ts](file:///c:/Users/asus/Downloads/coded/src/integrations/supabase/client.ts)
    *   *Why*: Connects all data grids directly to your databases.

## 📂 Step 3: Replace the Visual "Page" Files
Your external AI designer will output page code. Copy these new page files directly into their respective folders in this workspace:

| Target Page Location | Purpose | How to Connect to Logic |
| :--- | :--- | :--- |
| `src/pages/Homepage.tsx` | The bento grid landing page | Connect buttons to routes like `/rank-predictor`, `/cutoff-explorer` |
| `src/pages/Dashboard.tsx` | The central counseling hub timeline | Uses database metrics & timelines |
| `src/pages/RankPredictor.tsx` | Marks-to-rank calculator page | Import score calculator: `import { calculateRank } from "@/lib/rank-predictor"` |
| `src/pages/CutoffExplorer.tsx` | High-speed cutoff grid browser | Uses supabase database queries |
| `src/pages/CollegeFinder.tsx` | Matchfinder recommender results | Uses supabase database query filters |
| `src/pages/AICounselor.tsx` | Full-screen counselor chat wrapper | Import send assistant action: `import { sendMessage } from "@/lib/gemini"` |
| `src/pages/Donate.tsx` | Support UPI copying page | Uses `import.meta.env.VITE_UPI_ID` for dynamic copy |

## 💅 Step 4: Overwrite/Merge CSS Styles & Tailwind Configurations
If your external AI generator uses custom Tailwind classes or a specific styling theme:

1.  **Tailwind Configurations**:
    *   Open the generated `tailwind.config.js` or `tailwind.config.ts` from your AI designer.
    *   Replace or merge its contents with the existing `tailwind.config.js` in the root of this project.
2.  **Global CSS styling**:
    *   If the AI designer outputs a `global.css` or `index.css`:
    *   Open and overwrite [src/index.css](file:///c:/Users/asus/Downloads/coded/src/index.css) to adopt all new HSL colors, borders, custom layers, and styling tokens.

---

## 🗺️ Step 5: Update the Navigation & Router (`src/App.tsx`)
1.  Open [src/App.tsx](file:///c:/Users/asus/Downloads/coded/src/App.tsx) and ensure the new pages are correctly imported:
    ```tsx
    import Homepage from "./pages/Homepage";
    import Dashboard from "./pages/Dashboard";
    import RankPredictor from "./pages/RankPredictor";
    import CutoffExplorer from "./pages/CutoffExplorer";
    import CollegeFinder from "./pages/CollegeFinder";
    import AICounselor from "./pages/AICounselor";
    import Donate from "./pages/Donate";
    ```
2.  Ensure they map to the correct active `Route` paths inside the Router wrapper:
    ```tsx
    <Routes>
      <Route path="/" element={<Homepage />} />
      <Route path="/dashboard" element={<Dashboard />} />
      <Route path="/rank-predictor" element={<RankPredictor />} />
      <Route path="/cutoff-explorer" element={<CutoffExplorer />} />
      <Route path="/college-finder" element={<CollegeFinder />} />
      <Route path="/ai-counselor" element={<AICounselor />} />
      <Route path="/donate" element={<Donate />} />
    </Routes>
    ```

---

## ⚡ Step 6: Verify and Run Locally
Once the files are moved, open your terminal inside the root directory (`c:\Users\asus\Downloads\coded`) and run:

1.  **Verify compilation**:
    ```bash
    npm run build
    ```
    *This checks if there are any broken import references or missing TypeScript variables.*
2.  **Start your interactive development server**:
    ```bash
    npm run dev
    ```
    *Open the local port in your browser to interact with the stunning, newly merged frontend!*
